{# --- TAB DOCKERHUB --- #}
Despliegue de Imágenes Externas

Despliega cualquier imagen pública de DockerHub. PaaSify la descargará automáticamente.

Campo Valor/Regla Estado
image Ruta DockerHub (ej: python:3.9-slim) Obligatorio
environment Objeto JSON { "KEY": "VALUE" } Opcional
internal_port Puerto que usa la app (ej: 3000) Opcional
is_web Booleano (true/false) Opcional
container_type web, api, database, misc Opcional
keep_volumes Booleano (true/false) Opcional
project ID del proyecto (ej: 1) Obligatorio
subject ID de la asignatura (ej: 1) Obligatorio
¿Es necesario environment?

Solo si tu app requiere configs externas como credenciales de DB o claves de API.

Tip de Puerto

Si tu app no usa el puerto 80 por defecto, es obligatorio enviar internal_port.

{# --- TAB CUSTOM --- #}
Modo Personalizado (Custom Build)

Despliegue basado en código fuente (Archivo ZIP con Dockerfile) o mediante configuración (Archivo docker-compose.yml sin necesidad de ZIP).

Requisitos del Proyecto:

PaaSify detecta el modo de despliegue según los archivos en la raíz del ZIP.

Campo Valor/Regla Estado
mode Debe ser "custom" Obligatorio
code Archivo .zip (código fuente) Requerido con Dockerfile Opcional con Compose
project ID del proyecto (numérico) Obligatorio
subject ID de la asignatura (numérico) Obligatorio
container_configs Ajustes por contenedor (JSON)
Estructura: {"nombre": {"container_type": "...", "is_web": bool}}
Solo Compose
container_type Para modo Dockerfile: web, api, database, misc Opcional
is_web Para modo Dockerfile: true/false Opcional
environment Para modo Dockerfile: JSON con variables
(ej: {"API_URL": "...", "DEBUG": "1"}) 		Opcional
keep_volumes Conservar datos entre despliegues: true/false Opcional
Enviar dockerfile O compose Uno Obligatorio
Detección Automática

El sistema prioriza los archivos encontrados:

  • Si existe docker-compose.yml, se usa modo Compose.
  • Si no, buscará un Dockerfile (aquí el code.zip sí es obligatorio).

Nota: Si usas Compose, puedes omitir el parámetro code si no necesitas archivos adicionales para el build.

{# CASO 1: DOCKERFILE #}
📦 Proyecto con Dockerfile

Ideal para una aplicación única (React, Node.js, Python) que quieres que PaaSify compile de cero.

  • El ZIP debe tener el Dockerfile en la raíz.
  • Usa el método multipart/form-data.
CURL DOCKERFILE 
curl -X POST {{ PAASIFY_API_URL }}/containers/ \
  -H "Authorization: Bearer <TU_TOKEN>" \
  -F "name=mi-app-compilada" \
  -F "mode=custom" \
  -F "code=@proyecto.zip" \
  -F "dockerfile=@Dockerfile" \
  -F "project=1" \
  -F "subject=1" \
  -F "container_type=api" \
  -F "is_web=false"
{# CASO 2: COMPOSE #}
🏗️ Stack con Docker Compose

Para orquestar varios servicios (ej: Web + Base de Datos) o configurar volúmenes persistentes.

  • El ZIP debe tener el docker-compose.yml en la raíz.
  • Se respetarán los puertos y nombres definidos en el YAML.
CURL COMPOSE 
curl -X POST {{ PAASIFY_API_URL }}/containers/ \
  -H "Authorization: Bearer <TU_TOKEN>" \
  -F "name=mi-stack-completo" \
  -F "mode=custom" \
  -F "code=@bundle.zip" \          # Opcional si solo usas Compose ℹ️
  -F "compose=@docker-compose.yml" \
  -F "project=1" \
  -F "subject=1" \
  -F 'container_configs={"web":{"container_type":"web","is_web":true},"db":{"container_type":"database","is_web":false}}'
{# --- TAB EDIT (MODIFICAR) --- #}
Nota importante sobre Edición
Al modificar un servicio por API, PaaSify eliminará y recreará el contenedor para aplicar los cambios. Los servicios del catálogo oficial no permiten modificación.
Parámetros disponibles (PATCH)
Campo Descripción
env_vars JSON con variables de entorno
assigned_port Puerto externo (40000-50000)
internal_port Puerto interno de la app
is_web Boolean (true/false)
container_type web, api, database, misc
dockerfile Subida de archivo (multipart)
compose Subida de archivo (multipart)
code Subida de código .zip (multipart)
keep_volumes Boolean (true/false)
CURL PATCH 
curl -X PATCH {{ PAASIFY_API_URL }}/containers/{id}/ \
  -H "Authorization: Bearer <TU_TOKEN>" \
  -F "assigned_port=40500" \
  -F "is_web=true" \
  -F "keep_volumes=true" \
  -F 'env_vars={"DEBUG": "false", "PORT": "8080"}' \
  -F "dockerfile=@Nuevo.Dockerfile"

*Por defecto keep_volumes es true. Reemplaza {id} por el ID numérico de tu servicio.
