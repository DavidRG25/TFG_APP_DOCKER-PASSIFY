{# --- TAB DOCKERHUB --- #}
Despliegue de Imágenes Externas

Despliega cualquier imagen pública de DockerHub. PaaSify la descargará automáticamente.

CampoValor/ReglaEstado
imageRuta DockerHub (ej: python:3.9-slim)Obligatorio
environmentObjeto JSON { "KEY": "VALUE" }Opcional
internal_portPuerto que usa la app (ej: 3000)Opcional
¿Es necesario environment? Solo si tu app requiere configs externas (DB, API Keys).
Tip de Puerto: Si tu app no usa el puerto 80 por defecto, es obligatorio enviar el campo internal_port.
{# CASO 1: BÁSICO #}
📦 Uso Estándar

Ideal para imágenes que ya vienen configuradas (ej: nginx o bases de datos con valores por defecto).

DOCKERHUB BÁSICO 
curl -X POST {{ PAASIFY_API_URL }}/containers/ \
  -H "Authorization: Bearer " \
  -H "Content-Type: application/json" \
  -d '{
    "name": "mi-app",
    "image": "nginx:latest",
    "mode": "dockerhub"
  }'
{# CASO 2: CON VARIABLES #}
⚙️ Configuración Avanzada

Usa el campo environment para inyectar variables que tu app leerá al arrancar.

CON VARIABLES (ENV) 
curl -X POST {{ PAASIFY_API_URL }}/containers/ \
  -H "Authorization: Bearer " \
  -H "Content-Type: application/json" \
  -d '{
    "name": "app-python",
    "image": "python:3.9-slim",
    "mode": "dockerhub",
    "internal_port": 5000,
    "environment": {
      "DB_HOST": "10.5.0.1",
      "DEBUG": "true"
    }
  }'
{# --- TAB CUSTOM --- #}
Modo Personalizado (Custom Build)

Despliegue basado en código fuente empaquetado en un ZIP.

Requisitos del ZIP:

PaaSify detecta el modo según los archivos encontrados en la raíz del archivo ZIP.

modeDebe ser "custom"
fileRuta al archivo .zip (usa miltipart/form-data)
Detección automática: Si el ZIP contiene docker-compose.yml se usará Compose. Si no, buscará un Dockerfile.
{# CASO 1: DOCKERFILE #}
📦 Proyecto con Dockerfile

Ideal para una aplicación única (React, Node.js, Python) que quieres que PaaSify compile de cero.

  • El ZIP debe tener el Dockerfile en la raíz.
  • Usa el método multipart/form-data.
CURL DOCKERFILE 
curl -X POST {{ PAASIFY_API_URL }}/containers/ \
  -H "Authorization: Bearer " \
  -F "name=mi-app-compilada" \
  -F "mode=custom" \
  -F "file=@proyecto.zip"
{# CASO 2: COMPOSE #}
🏗️ Stack con Docker Compose

Para orquestar varios servicios (ej: Web + Base de Datos) o configurar volúmenes persistentes.

  • El ZIP debe tener el docker-compose.yml en la raíz.
  • Se respetarán los puertos y nombres definidos en el YAML.
CURL COMPOSE 
curl -X POST {{ PAASIFY_API_URL }}/containers/ \
  -H "Authorization: Bearer " \
  -F "name=mi-stack-completo" \
  -F "mode=custom" \
  -F "file=@bundle.zip"
